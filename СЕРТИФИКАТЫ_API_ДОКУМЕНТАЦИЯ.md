# Документация API сертификатов

## Обзор
Этот документ описывает конечные точки REST API для системы сертификатов в образовательной платформе. Система позволяет студентам получать сертификаты после завершения курсов, загружать их в формате PDF и проверять их подлинность.

## Базовый URL
`/api/certificates`

## Аутентификация
Большинство конечных точек требуют аутентификации через JWT токен:
```
Authorization: Bearer <token>
```

## Контроль доступа по ролям
- **Студенты**: Могут создавать, просматривать и загружать свои собственные сертификаты
- **Инструкторы**: Могут просматривать сертификаты для своих курсов
- **Администраторы**: Полный доступ ко всем сертификатам (включая отзыв и продление)

---

## Конечные точки

### 1. Создание сертификата
**POST** `/api/certificates/generate`

**Доступ**: Приватный (только студенты)

**Описание**: Создает сертификат для завершенного курса

**Тело запроса**:
```json
{
  "courseId": "course_object_id",
  "grade": "A+", // Необязательно: A+, A, A-, B+, B, B-, C+, C, C-, D, F
  "score": 95.5, // Необязательно: процентный балл (0-100)
  "metadata": { // Необязательно: дополнительные данные о завершении курса
    "totalDuration": 20,
    "totalLessons": 15,
    "completedLessons": 15,
    "completionPercentage": 100
  }
}
```

**Ответ**:
```json
{
  "success": true,
  "message": "Сертификат успешно создан",
  "data": {
    "certificateId": "CERT-123456789-ABCDEF",
    "verificationCode": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
    "filePath": "/certificates/certificate_user_course_timestamp.pdf",
    "certificate": {
      "_id": "certificate_object_id",
      "certificateId": "CERT-123456789-ABCDEF",
      "userId": "user_object_id",
      "courseId": "course_object_id",
      "courseTitle": "Название курса",
      "userName": "Иван Петров",
      "userEmail": "ivan@example.com",
      "issueDate": "2024-01-01T00:00:00.000Z",
      "grade": "A+",
      "score": 95.5,
      "status": "active",
      "filePath": "/certificates/certificate_user_course_timestamp.pdf",
      "verificationCode": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
      "metadata": {
        "totalDuration": 20,
        "totalLessons": 15,
        "completedLessons": 15,
        "completionPercentage": 100
      },
      "createdAt": "2024-01-01T00:00:00.000Z",
      "updatedAt": "2024-01-01T00:00:00.000Z"
    }
  }
}
```

**Ошибки**:
- 400: Ошибка валидации, курс не завершен, сертификат уже существует
- 403: Не записан на курс
- 404: Курс не найден

### 2. Получить мои сертификаты
**GET** `/api/certificates/my-certificates`

**Доступ**: Приватный

**Описание**: Возвращает все сертификаты для аутентифицированного пользователя

**Параметры запроса**:
- `page` (число): Номер страницы (по умолчанию: 1)
- `limit` (число): Элементов на странице (по умолчанию: 10)

**Ответ**:
```json
{
  "success": true,
  "count": 3,
  "total": 5,
  "page": 1,
  "pages": 2,
  "data": [
    {
      "_id": "certificate_id",
      "certificateId": "CERT-123456789-ABCDEF",
      "courseId": {
        "_id": "course_id",
        "title": "Название курса"
      },
      "userName": "Иван Петров",
      "issueDate": "2024-01-01T00:00:00.000Z",
      "grade": "A+",
      "score": 95.5,
      "status": "active",
      "createdAt": "2024-01-01T00:00:00.000Z"
    }
  ]
}
```

    "userId": {

    "courseTitle": "Название курса",
    "userName": "Иван Петров",
    "userEmail": "ivan@example.com",
    "issueDate": "2024-01-01T00:00:00.000Z",
    "expiryDate": null,
    "grade": "A+",
    "score": 95.5,

**Контроль доступа**:
- Владельцы сертификатов могут получить доступ к своим сертификатам
- Инструкторы курсов могут получить доступ к сертификатам для своих курсов
- Администраторы могут получить доступ ко всем сертификатам

### 4. Загрузить PDF сертификата
**GET** `/api/certificates/download/:id`

**Доступ**: Приватный

**Описание**: Загружает PDF файл сертификата

**Примечание**: `:id` может быть как ID сертификата, так и MongoDB object ID

**Ответ**: Загрузка PDF файла

**Контроль доступа**:
- Владельцы сертификатов могут загрузить свои сертификаты
- Инструкторы курсов могут загрузить сертификаты для своих курсов
- Администраторы могут загрузить все сертификаты

---

### 5. Проверить сертификат
**GET** `/api/certificates/verify/:certificateId`

**Доступ**: Публичный

**Описание**: Проверяет подлинность и действительность сертификата

**Примечание**: `:certificateId` может быть как ID сертификата, так и код проверки

**Ответ**:
```json
{
  "success": true,
  "valid": true,
  "certificate": {
    "id": "CERT-123456789-ABCDEF",
    "verificationCode": "verification_code",
    "userId": "user_id",
    "userName": "Иван Петров",
    "userEmail": "ivan@example.com",
    "courseId": "course_id",
    "courseTitle": "Название курса",
    "issueDate": "2024-01-01T00:00:00.000Z",
    "expiryDate": null,
    "grade": "A+",
    "score": 95.5,
    "status": "active"
  },
  "message": "Сертификат действителен"
}
```

**Логика проверки**:
- Проверяет существование сертификата
- Проверяет статус (действителен против отозван/просрочен)
- Проверяет дату окончания, если установлена
- Возвращает подробную информацию о сертификате

### 7. Отозвать сертификат
**PUT** `/api/certificates/revoke/:id`

**Доступ**: Приватный (только администраторы)

**Описание**: Отзывает сертификат, помечая его как недействительный

**Тело запроса**:
```json
{
  "reason": "Причина отзыва"
}
```

**Ответ**:
```json
{
  "success": true,
  "message": "Сертификат успешно отозван",
  "data": {
    // Обновленный объект сертификата со статусом: 'revoked'
  }
}
```

---

### 8. Продлить срок действия сертификата
**PUT** `/api/certificates/extend/:id`

**Доступ**: Приватный (только администраторы)

**Описание**: Продлевает дату окончания действия сертификата

**Тело запроса**:
```json
{
  "expiryDate": "2025-12-31T23:59:59.000Z"
}
```

**Ответ**:
```json
{
  "success": true,
  "message": "Срок действия сертификата успешно продлен",
  "data": {
    // Обновленный объект сертификата с новой датой окончания
  }
}
```

## Модель данных

### Схема сертификата
```javascript
{
  certificateId: String (уникальный, генерируется автоматически),
  userId: ObjectId (ссылка: User, обязательный),
  courseId: ObjectId (ссылка: Course, обязательный),
  courseTitle: String (обязательный),
  userName: String (обязательный),
  userEmail: String (обязательный),
  issueDate: Date (по умолчанию: сейчас),
  expiryDate: Date (необязательный),
  grade: String (значения: A+ до F),
  score: Number (0-100),
  status: String (значения: active, revoked, expired, по умолчанию: active),
  filePath: String (путь к PDF файлу),
  verificationCode: String (уникальный, UUID),
  metadata: {
    totalDuration: Number,
    totalLessons: Number,
    completedLessons: Number,
    completionPercentage: Number
  },
  timestamps: true
}
```

## Функция автоматического создания

Сертификаты автоматически создаются, когда студент завершает курс (достигает 100% прогресса). Система:

1. Обнаруживает изменение прогресса с <100% до 100%
2. Определяет оценку на основе итогового балла
3. Создает запись сертификата в базе данных
4. Генерирует PDF файл
5. Связывает сертификат с пользователем и курсом

**Назначение оценки**:
- 95-100%: A+
- 90-94%: A
- 85-89%: A-
- 80-84%: B+
- 75-79%: B
- 70-74%: B-
- 65-69%: C+
- 60-64%: C
- Ниже 60%: F

## Обработка ошибок

Все конечные точки возвращают соответствующие коды состояния HTTP:

- `200` - Успешно
- `201` - Создано (сертификат создан)
- `400` - Неверный запрос (ошибки валидации, нарушения бизнес-логики)
- `401` - Не авторизован (нет токена)
- `403` - Запрещено (недостаточно прав)
- `404` - Не найдено
- `500` - Внутренняя ошибка сервера

Стандартный ответ об ошибке:
```json
{
  "success": false,
  "message": "Описание ошибки",
  "errors": [...] // необязательно, для ошибок валидации
}
```

## Пример использования с cURL

```bash
# Создать сертификат
curl -X POST http://localhost:5000/api/certificates/generate \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"courseId": "course_id", "grade": "A+", "score": 95}'

# Получить мои сертификаты
curl http://localhost:5000/api/certificates/my-certificates \
  -H "Authorization: Bearer <token>"

# Загрузить сертификат
curl http://localhost:5000/api/certificates/download/CERT-123456789-ABCDEF \
  -H "Authorization: Bearer <token>" \
  -O

# Проверить сертификат (публичная конечная точка)
curl http://localhost:5000/api/certificates/verify/CERT-123456789-ABCDEF

# Получить сертификаты курса
curl http://localhost:5000/api/certificates/course/course_id \
  -H "Authorization: Bearer <token>"
```

## Процесс проверки

Внешние стороны могут проверить подлинность сертификата:
1. Используя публичную конечную точку проверки
2. Предоставляя ID сертификата или код проверки
3. Получая подтверждение действительности и подробную информацию о сертификате

Это позволяет работодателям, учебным заведениям или другим сторонам независимо проверять подлинность сертификата без доступа к платформе напрямую.
